# CS 300 Data Structures and Algorithms Portfolio

## Reflection

The problem I was solving in the projects for this course was how to store, organize, sort, and display course information for a Computer Science advising program. The program needed to read course data, store each course with its course number, title, and prerequisites, and then print the courses in alphanumeric order. The goal was to design a program that could help users view course information clearly and efficiently.

I approached the problem by comparing different data structures, including vectors, hash tables, and binary search trees. Understanding data structures is important because the structure chosen affects how efficiently a program can store, search, sort, and display information. For this project, it was important to choose a structure that made sense for organizing course data and printing the course list in order.

When I ran into roadblocks, I worked through them step by step. Some of the harder parts were understanding how to load the course data correctly, how to connect prerequisites to the correct courses, and how to make sure the output printed in the right order. I overcame these issues by breaking the program into smaller parts, testing each part, and checking the logic before moving on.

This project expanded my approach to designing software because it showed me the importance of planning before coding. Instead of just starting with code, I had to think about the problem, compare possible data structures, and decide which design would best meet the requirements. That helped me understand how design decisions affect the final program.

My work on this project also changed the way I write programs so they are more maintainable, readable, and adaptable. I learned to use clearer function names, organize code into smaller sections, and write code that can be updated more easily later. This is important because real software often needs to be changed, improved, or reused, and readable code makes that much easier.
